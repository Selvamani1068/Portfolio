Create developer documentation in Markdown

MadCap Flare supports Markdown (.md) topic files natively alongside XML-based .htm topics. This section shows you how to create a Markdown topic file, apply the correct naming conventions, write content that follows the Microsoft Writing Style Guide, and add the topic to the table of contents.

Step 1: Create a new Markdown topic file

To add a Markdown topic to your Flare project:

  1. Open your Flare project.
  2. In the Content Explorer panel on the left, right-click the folder where you want to create the new file. For example, right-click the Content/Topics folder.
  3. Select Add > New Topic. The Add File dialog opens.
  4. In the File Type list, select Markdown.
  5. In the File Name field, enter a name using kebab-case. For example: authentication-guide.md
  6. Leave the template as the default blank template, or select a custom template if one is available.
  7. Select Add. Flare creates the file and opens it in the text editor.

Note  Native Markdown support requires MadCap Flare 2019 R2 (v15.1) or later. If your version is older, upgrade before following these steps.

Step 2: Apply file naming conventions

Consistent file names make the project easier to navigate and prevent broken links when topics are reorganized. Follow these rules when naming Markdown files:

Rule Correct example Incorrect example
Use lowercase letters only api-reference.md API_Reference.md
Separate words with hyphens (kebab-case) getting-started.md getting_started.md
Use a descriptive name that reflects the topic configure-webhooks.md topic1.md
Group related files with a shared prefix auth-overview.md, auth-tokens.md overview.md, tokens.md
Avoid spaces and special characters rate-limits.md rate limits (v2).md

Step 3: Write your documentation content

Open the .md file in Flare's text editor and write your content. The following structure is recommended for most developer documentation topics: 

# Topic title

Short description: one or two sentences that summarize what the topic covers.

## Prerequisites

List what the reader needs before they start.

## Step 1: First action

Describe what to do. Start the heading with an imperative verb.

1. Do the first thing.
2. Do the second thing.
3. Do the third thing.

## Step 2: Next action

Continue the procedure.

## Next steps

Link to the next topic in the workflow.

Microsoft Writing Style Guide tip  Use the imperative mood in headings and step text: write Configure the webhook, not Configuring the webhook or How to configure the webhook. Be direct and use second person: write you, not the user.

The following table shows how Flare renders the most common Markdown elements:

Element Markdown syntax When to use it
Topic title # Title One per file; used as the topic title in the TOC and breadcrumb
Section heading ## Heading Major sections within a topic
Subsection heading ### Heading Sub-steps or sub-sections within a major section
Numbered list 1. Item Sequential steps in a procedure
Bulleted list - Item Items without a required order, such as feature lists or prerequisites
Code block ```bash … ``` Command-line instructions, code samples, or configuration file content
Inline code `code` File names, commands, parameters, and UI element names in body text
Note callout > **Note** … Supplementary information that is not critical to the main procedure
Warning callout > **Important** … Information that can cause data loss or unexpected behavior if ignored
Hyperlink [text](relative-path.md) Cross-topic links; use relative paths for topics within the same project

Step 4: Add the topic to the table of contents

A Markdown topic is not automatically included in your Flare output. You must add it to the Table of Contents (TOC) file before it appears in the published WebHelp:

  1. In the Project Organizer, expand TOCs and double-click the .fltoc file for your target. The TOC Editor opens.
  2. In the Content Explorer, locate the .md file you created.
  3. Drag the .md file from the Content Explorer into the TOC Editor at the position where you want it to appear.
  4. Alternatively, right-click the TOC node above the intended position and select Link to Topic. Browse to and select the .md file.
  5. Save the TOC file: press Ctrl+S.
  6. Commit the TOC change to Git along with the new topic file (see Branching strategy for commit steps).

Markdown limitations in Flare

The following Flare features are not available in .md topic files. If your topic requires any of these features, create it as an .htm file instead:

  • Conditions — MadCap:conditions attributes cannot be applied to Markdown elements.
  • Snippets — MadCap:snippetBlock and MadCap:snippetText are not supported in Markdown files.
  • Variables — MadCap:variable tags are not processed in Markdown files.
  • Cross-references — Use standard Markdown hyperlinks instead of MadCap:xref.

Important  Do not embed Flare XML tags in a Markdown file. Flare does not process them and outputs them as literal text, which corrupts the published topic.
