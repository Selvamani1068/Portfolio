Link a Flare project to a Git repository

Bind your MadCap Flare project to a Git remote repository by using Flare's built-in Source Control integration. After you bind the project, authors can commit, pull, push, and create branches without leaving the Flare IDE.

Before you begin

Confirm that the following conditions are met before you bind the project:

  • Git is installed and available on the system PATH (run git --version to verify).
  • The Flare project has been saved to a local folder.
  • The local project folder does not already contain a .git directory.
  • The remote repository exists and you have write access to it.
  • You have completed the steps in Set up Git for a Flare project.

Important  Do not bind a Flare project that already has a .git folder in the project root. Binding overwrites the existing Git configuration and can cause history loss. If a repository already exists, use Source Control > Get Latest to connect instead.

Bind the Flare project to Git

To bind a MadCap Flare project to a Git repository, follow these steps:

  1. Open the Flare project.
  2. On the menu bar, select Source Control > Bind Project. The Bind Project dialog opens.
  3. In the Source Control list, select Git.
  4. In the Remote field, enter the HTTPS or SSH URL of the remote repository.
  5. Enter your credentials (username and personal access token, or SSH key passphrase).
  6. Select OK. Flare initializes a local Git repository in the project root, stages all project files, and performs an initial commit.
  7. When prompted to push, select Push to upload the initial commit to the remote repository.

Note  Flare uses the system Git installation for all source-control operations. If Flare cannot detect Git, verify that the Git binary folder is on the system PATH and restart Flare.

Verify the binding

After binding, confirm that the integration is working correctly:

  1. Select Source Control > View History. The initial commit should appear in the history list.
  2. Open a topic file and make a minor edit (for example, add a space). The file icon in the Content Explorer should display a pending-change indicator.
  3. Select Source Control > Check In, enter a commit message, and select Check In. Confirm that the commit appears in View History.

Daily workflow commands

The following table maps common daily tasks to their Flare menu path and the equivalent Git command-line operation:

Task Flare menu path Equivalent Git command
Pull the latest changes from the remote Source Control > Get Latest git pull
Stage and commit changes Source Control > Check In git add . && git commit -m "message"
Push commits to the remote Source Control > Push git push
Create a new branch Source Control > Branch > Create Branch git checkout -b branch-name
Switch to an existing branch Source Control > Branch > Switch Branch git checkout branch-name
View file change history Source Control > View History git log --oneline
Discard local changes Source Control > Undo Check Out git checkout -- file

Troubleshoot common issues

Symptom Likely cause Resolution
Flare cannot detect Git Git not on system PATH Add the Git binary folder to PATH and restart Flare
Authentication fails on push Expired personal access token Generate a new token in the remote repository settings and update credentials
Source Control menu is unavailable Project not bound to source control Select Source Control > Bind Project and complete the binding steps
