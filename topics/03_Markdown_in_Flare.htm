Use Markdown in MadCap Flare

MadCap Flare supports Markdown (.md) topic files natively alongside XML-based .htm topics. This allows developer contributors to write content in familiar Markdown syntax while Flare handles compilation and output generation.

Overview

When you add a .md file to a Flare project, Flare treats it as a first-class topic. The file appears in the Content Explorer, can be added to the Table of Contents (TOC), and is compiled into the output alongside .htm topics. Flare converts the Markdown syntax to HTML at build time.

Note  Native Markdown support requires MadCap Flare 2019 R2 (v15.1) or later. Earlier versions require a third-party preprocessing tool.

Add a Markdown topic

To add a new Markdown topic to a Flare project, follow these steps:

  1. In the Content Explorer, right-click the folder where you want to create the file.
  2. Select Add > New Topic.
  3. In the Add File dialog, set File Type to Markdown.
  4. Enter a filename by using kebab-case—for example, getting-started.md.
  5. Select a template if one is available, or leave the default blank template.
  6. Select Add. Flare opens the file in the built-in text editor.

Note  Markdown files are stored in the Content/ folder alongside .htm topics. Use consistent folder organization for both file types to keep the project maintainable.

File naming conventions

Follow these conventions when you name Markdown topic files:

  • Use kebab-case: api-reference.md, not API_Reference.md.
  • Use descriptive, lowercase names that reflect the topic content.
  • Avoid spaces, underscores, and special characters in filenames.
  • Prefix related files with a shared identifier: auth-overview.md, auth-tokens.md.

Supported Markdown elements

Flare supports the CommonMark specification for Markdown. The following table describes how key Markdown elements map to HTML output:

Element Markdown syntax Output (HTML) Notes
Heading 1 # Title <h1> Used as the topic title in the TOC and breadcrumb
Headings 2–4 ## / ### / #### <h2>–<h4> Maps to the corresponding heading tag
Fenced code block ```lang … ``` <pre><code class="lang"> Specify language for syntax highlighting
Inline code `code` <code> Use for UI elements, file names, and commands
Blockquote callout > **Note:** … <blockquote> Style by using the DevDocs.css stylesheet
Unordered list - item <ul><li> Use for items without a required sequence
Ordered list 1. item <ol><li> Use for procedures and sequential steps
Table Pipe-delimited rows <table> Flare applies the project table style automatically
Hyperlink [text](url) <a href="url"> Use relative paths for cross-topic links

Limitations

The following Flare features are not available in .md topic files. Use standard .htm topics for content that requires these features:

  • Conditions — MadCap:conditions attributes cannot be applied to Markdown elements.
  • Snippets — MadCap:snippetBlock and MadCap:snippetText are not supported.
  • Variables — MadCap:variable tags cannot be inserted in Markdown files.
  • Cross-references — MadCap:xref is not available; use standard Markdown hyperlinks instead.

Important  If a topic requires conditions, snippets, or variables, create it as an .htm file. Do not attempt to embed Flare XML tags inside a Markdown file—Flare does not process them and outputs them as literal text.

Add the Markdown topic to the TOC

A Markdown topic is not automatically included in the output. You must add it to the TOC manually:

  1. Open the project TOC file (.fltoc) in the TOC Editor.
  2. Drag the .md file from the Content Explorer into the desired position in the TOC.
  3. Alternatively, right-click the target TOC node and select Link to Topic, then browse to the .md file.
  4. Save the TOC file and commit the change to Git.
